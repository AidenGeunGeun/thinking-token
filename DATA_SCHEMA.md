# Data Schema

This project does not define a custom trajectory format. tau2-bench remains the source of truth for run outputs.

## Files Saved Per Run

Each run directory contains:

- tau2-bench's standard trajectory output (`results.json` and related artifacts)
- `thinking_analysis.jsonl`, a lightweight per-turn analysis file derived from the trajectory

## Directory Layout

```text
results/
└── phase1/
    └── {model}_{condition}_{timestamp}/
        ├── results.json
        ├── thinking_analysis.jsonl
        └── summary.json
```

## `thinking_analysis.jsonl`

One JSON object per turn.

```json
{
  "task_id": "telecom_001",
  "trial": 0,
  "turn_index": 3,
  "retention_strategy": "window_3",
  "assistant_message_count": 2,
  "raw_thinking_chars": 8120,
  "raw_thinking_tokens_approx": 2030,
  "summary_chars": 420,
  "summary_tokens_approx": 105,
  "summarizer_input_tokens": 2030,
  "summarizer_output_tokens": 105,
  "retained_at_end": true,
  "window_size": 3,
  "prompt_tokens_total": 14200,
  "prompt_tokens_cached": 12800,
  "prompt_tokens_evaluated": 1400,
  "generation_tokens": 3200,
  "thinking_tokens_in_generation": 2800,
  "source": "tau2 trajectory"
}
```

### Field Notes

- `task_id`: tau2-bench task identifier
- `trial`: zero-based trial index
- `turn_index`: zero-based user turn index
- `retention_strategy`: `strip_all`, `window_3`, or `retain_all`
- `assistant_message_count`: number of assistant messages observed in the turn
- `raw_thinking_chars`: total extracted `<think>` text length for the turn (before summarization)
- `raw_thinking_tokens_approx`: rough token estimate using `len(text) // 4`
- `summary_chars`: length of summarized thinking (null for strip_all and thinking_off)
- `summary_tokens_approx`: rough token estimate for summary
- `summarizer_input_tokens`: tokens sent to summarizer model (null if no summarization)
- `summarizer_output_tokens`: tokens generated by summarizer model (null if no summarization)
- `retained_at_end`: whether this turn's thinking summary is still retained in the terminal conversation state after the run completes
- `window_size`: integer for windowed strategies, otherwise `null`

### Cache Accounting Fields

These fields track the real compute cost per LLM call, which varies by retention strategy:

- `prompt_tokens_total`: total tokens in the prompt sent to the model
- `prompt_tokens_cached`: tokens that hit KV cache (prefix match from previous turn). Only new tokens past this point require computation.
- `prompt_tokens_evaluated`: tokens that required fresh computation (`total - cached`)
- `generation_tokens`: total output tokens (thinking + response)
- `thinking_tokens_in_generation`: thinking tokens within the generation output

**Why this matters**: `strip_all` and `retain_all` are append-only histories (perfect cache reuse). `window_3` mutates history each turn as the window slides, invalidating the cache at the mutation point. The `prompt_tokens_evaluated` field captures this difference directly.

**Note**: Whether llama-server reports cache hits in its response is TBD. If not available from the server, these fields will be computed by diffing consecutive prompts to find the divergence point.

- `source`: provenance marker for downstream analysis

`retained_at_end` reflects terminal state only. It does not mean the turn was unavailable to every later prompt; for windowed retention, turns can be retained for some intermediate prompts and later fall out of the window. Actual per-prompt retention is enforced at each LLM call by `src/thinking.py`'s `apply_retention_strategy()`.

The analysis file is intentionally narrow: it complements tau2-bench trajectories instead of replacing them.

## Checkpoint / Resume

`run_phase1.py` supports checkpoint/resume. A condition run is "complete" when its `summary.json` exists with matching `task_ids` and `num_simulations`. On restart, completed conditions are skipped automatically. Use `--fresh` to ignore existing results and re-run everything.

Partial run directories (interrupted mid-condition, no `summary.json`) are cleaned up automatically. First Ctrl+C requests graceful shutdown after the current condition; second Ctrl+C force-quits.

## Data Source

Thinking metrics in `thinking_analysis.jsonl` come from the **agent-side accumulator** (`src/agent._thinking_records`), not from `results.json`. The agent captures raw thinking chars, summary chars, and summarizer token usage from litellm *before* stripping thinking for the public view. The runner merges these agent records with trajectory turn structure after each condition run.

`results.json` still contains only the user-sim-visible view (clean text). The agent accumulator is the authoritative source for thinking token counts.

### Known Issues (Phase 1 historical data)

Phase 1 data collected before the accumulator fix has all-zero thinking metrics. This affects runs in `results/phase1/` from the 2B and 9B experiments. The fix is forward-only — re-running produces correct data.
