# thinking-tokens

## What This Project Is

Research project testing whether retaining *summarized* thinking tokens in conversation history improves multi-turn agent performance on τ²-bench telecom tasks. Anthropic proved full thinking preservation helps (Opus 4.5+), but that requires proprietary encrypted replay. Open-source models (Qwen3.5) strip thinking by default. This project tests whether lightweight summarization — via a cheap external model — can bridge that gap.

## Environment

- **Local (2B/4B)**: Apple M4 Pro 48GB — llama.cpp Metal
- **Cloud (9B)**: RunPod L40S 48GB — llama.cpp CUDA (`-DCMAKE_CUDA_ARCHITECTURES="89"`)
- **Python**: 3.12+ (via `uv`, venv at `~/thinking-tokens-venv` on Mac or `/workspace/venv` on RunPod)
- **Inference**: llama.cpp + GGUF Q8_0 weights, Q8_0 KV cache
- **User sim + summarizer**: MiMo V2 Flash via OpenRouter (`openrouter/xiaomi/mimo-v2-flash`)

## Required Environment Variables

```bash
export OPENROUTER_API_KEY=sk-or-...   # user sim + summarizer
export HF_TOKEN=hf_...                # model downloads
export LLAMA_SERVER_BIN=~/llama.cpp/build/bin/llama-server
export TAU2_DATA_DIR=~/tau2-repo/data  # NOTE: must be /data subdir, not repo root
```

`TAU2_DATA_DIR` pointing at the repo root (not `/data`) is a common mistake — tau2 looks for `$TAU2_DATA_DIR/tau2/domains/telecom/tasks.json`.

### Two-View Architecture (critical)

The agent maintains two separate message lists:
1. **`_internal_messages`** — contains `<think>` or `<think_summary>` blocks. Used to build the agent's LLM prompt.
2. **`state.messages`** (tau2 shared history) — ALL thinking stripped before returning. What the user simulator sees.

The retention strategy only operates on `_internal_messages`. The user sim always sees clean text. Violating this caused catastrophic contamination in the Phase 1 pilot (see EXPERIMENT.md).

### 6 Conditions

| Condition | enable_thinking | retention | summarize |
|---|---|---|---|
| `thinking_off` | false | strip | false |
| `strip_all` | true | strip | false |
| `raw_window3` | true | window_3 | false |
| `raw_retain` | true | retain_all | false |
| `summary_window3` | true | window_3 | true |
| `summary_retain` | true | retain_all | true |

## Commands

```bash
# Setup (run once)
bash scripts/setup_mac.sh           # Mac
bash scripts/setup_runpod.sh        # RunPod

# Select tasks (run once; deterministic)
python scripts/select_tasks.py

# Verify pipeline before any real run (all 6 conditions, 1 task, ~10 min)
python scripts/verify_pipeline.py --model qwen35-2b

# Targeted inspection: compare agent vs user-sim view for a condition
python scripts/view_conversation.py --both summary_retain --full-thinking
python scripts/view_conversation.py --both raw_retain --full-thinking

# Extended verify for qualitative comparison (natural termination)
python scripts/verify_pipeline.py --model qwen35-2b --condition raw_retain --max-steps 0
python scripts/verify_pipeline.py --model qwen35-2b --condition summary_retain --max-steps 0

# Dry-run and real runs
python scripts/run_phase1.py --dry-run
python scripts/run_phase1.py --model qwen35-2b
python scripts/run_phase1.py --model qwen35-4b
python scripts/run_phase1.py --model qwen35-9b   # RunPod only

# View results
python scripts/view_results.py
python scripts/view_results.py --detail

# Tests
python -m pytest tests/ -x
```

## Conventions

- Never mutate `_internal_messages` in place when applying retention; always deep-copy.
- Thinking artifacts (`<think>`, `<think_summary>`) MUST NOT appear in messages returned to tau2.
- Summary format: first-person stream-of-consciousness ("I checked their account..."), no headers or bullets.
- `select_tasks.py` is deterministic — same 10 tasks every run, no random seed needed.
- Log everything; results are expensive to regenerate. Raw outputs and thinking analysis are saved per run.
- Keep tau2 integration thin — compose around `LLMAgent`, don't fork framework logic.
